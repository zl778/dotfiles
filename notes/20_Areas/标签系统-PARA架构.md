#p/current

# 标签系统 — PARA + 十进制 架构

## 核心理念

- **P**rojects — 有截止日期的项目任务
	- - 有截止日期、有完成标准、完成以后就结束。
- **A**reas — 需长期维护标准的领域
	- 没有结束时间。
- **R**esources — 持续感兴趣的主题
	- 将来可能会用到，但目前没有正在做。
- **A**rchives — 已归档的非活跃条目
	- 所有结束、不再维护的内容。

信息的定义
	信息有三种，转化路径各不同
		• 备查类：知识储备、避免干扰
		•消化类：投入精力、注重转化
		•拓展类：已经内化、持续生长


标签前缀: `#p/` `#a/` `#r/` `#z/`
文件夹: `10_Projects/` `20_Areas/` `30_Resources/` `90_Archives/` `00_Inbox`

收件箱: `00_Inbox/`
核心：让信息既不被遗忘，又不干扰主线
可能有价值，暂不加工；
记录来源，标签待入库。

## 6. 使用规则

1. **一笔记一核心标签** — 主要归属一个标签即可，最多不超过 3 个
2. **标签 + 文件夹双轨** — 文件放对应文件夹，标签做正交检索
3. **项目标签用完即删** — 项目结束后移除 `#p/`，如需保留笔记改标 `#z/`
4. **不要造未登记的标签** — 新标签先加入本表再使用
5. **深度 ≤ 2 层** — `#a/work/low-current` 可，`#a/work/low-current/视频监控` 不可（用笔记标题区分）
6. **MOC 索引** — 每个大类建一个 MOC 笔记聚合子项（如 `20 Areas/_MOC.md`）
